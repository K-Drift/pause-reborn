# 1 SPEC.md — AI 暂停广告：腾讯视频暂停体验重塑系统
## 1.1 项目概述

**项目代号**:Pause Reborn(暂停重生)
**一句话定义**:把腾讯视频"暂停广告"从打断观影的弹窗,重塑为感知场景、理解用户、懂得克制的智能暂停体验系统。

---

## 1.2 核心愿景与产品立场

### 1.2.1 我们要解决的真实问题
腾讯视频现行的暂停广告存在三大用户痛点:
1. **内容无关**:广告与正在观看的剧情、情绪、画面毫无关联,造成认知断裂。
2. **形式粗暴**:弹窗居中、自动播放、无关闭按钮,破坏沉浸感。
3. **缺乏克制**:在任何场景任何时机硬性插入,无视用户当前心境。

### 1.2.2 我们的产品立场
**这不是一个广告优化工具,而是一个"暂停时刻"的体验系统。**
广告只是这个系统的输出之一。系统也可以输出"不打扰"、"内容推荐"、"情绪安抚"。
**核心信念**:广告不应是观影的中断,而应是观影的延续。

## 1.3 用户演示流程(Demo 主线)
按时间顺序,用户在 Demo 中会看到的完整体验:

### 1.3.1 主场景:原生广告生成

1. 用户打开网页,看到一个模拟腾讯视频的播放器,循环播放预置的剧集片段(如《人之初》某集)。
2. 用户点击暂停按钮。
3. 屏幕进入"对比模式",左右分屏:
    - **左侧(现状)**:展示腾讯视频真实暂停广告的截图(花花绿绿的弹窗)。
    - **右侧(AI 改造后)**:先展示加载动画"AI 正在分析画面…",3-5 秒后呈现 AI 生成的原生广告——色调与剧情一致、文案像旁白而非促销、形态融入画面而非遮挡。
4. 右侧调试面板同步展示 AI 的"思考过程":VLM 输出的场景 JSON、品牌选择理由、声线选择、契合度评分。

### 1.3.2 进阶场景:对话式追问

1. AI 广告生成后,用户可点击或语音追问(如"这酒多少钱?""有没有更便宜的同类?")。
2. AI 用与场景情绪同调的预设声线(不是剧中角色声音)回答。
3. 用户可继续追问或表达购买意向,完成"展示 → 沟通 → 转化"的完整闭环。

### 1.3.3 切换场景:用户画像分层

1. Demo 顶部提供"用户画像切换器",预设 3-4 个用户画像(如"35岁金融男"、"25岁年轻女生"、"40岁女性高管")。
2. 同一暂停帧,切换不同画像,AI 给出不同的具体品牌推荐。
3. 演示"画面定品类、画像定品牌"的分层决策逻辑。

### 1.3.4 反例场景:敏感场景识别

1. 切换到一个情绪沉重的暂停帧(如葬礼、病房、严重冲突场景)。
2. AI 主动判断不出任何广告,仅展示极简的纯黑暂停界面与一句克制的提示语。
3. 调试面板说明判断理由。

### 1.3.5 进阶场景:跨剧集情绪追踪

1. Demo 顶部提供"用户状态切换器",预设两种状态:
    - **状态A:情绪正常** — 同一暂停帧,正常匹配广告
    - **状态B:情绪疲劳累积** — 同一暂停帧,AI 不再推广告,而是推荐"切换到轻喜剧《武林外传》"
2. 演示系统具备"平台视角"的关怀能力。

## 1.4 技术架构

### 1.4.1 三段式核心 Pipeline

```
[暂停帧截图]
    ↓
【第一段】场景理解(VLM)
   输入:暂停帧图像
   输出:结构化场景 JSON(场景、情绪、色调、物体、空白表面)
    ↓
【第二段】广告匹配 + 文案生成 + 声线选择(LLM)
   输入:场景 JSON + 用户画像 + 用户状态 + 广告库
   输出:选中品牌 + 情境化文案 + 推荐声线 + 决策理由 + 评分
    ↓
【第三段】原生广告视觉生成(GPT-Image-2)
   输入:暂停帧 + 品牌信息 + 文案 + 风格指令
   输出:色调统一、文字精准、融入画面的原生广告图
    ↓
[最终 AI 改造后的暂停广告]
```

### 1.4.2 三层决策模型

每次暂停的决策由三个维度叠加:

|决策层|信号来源|决定什么|
|---|---|---|
|画面层(实时)|VLM 对当前帧的理解|这一帧适合什么品类|
|用户层(静态)|预设的用户画像|在该品类里推哪个具体品牌/SKU|
|状态层(动态)|用户暂停历史的情绪累积|此刻是否该出广告 / 是否切换为陪伴模式|

**特殊规则**:

- 状态层可以**否决**画面层(如孕妇画像 + 酒吧场景 → 不推酒)
- 状态层可以**改变输出形态**(情绪疲劳 → 不出广告,改为内容推荐)

### 1.4.3 敏感场景识别

在第一段 VLM 输出中增加 `sensitivity` 字段(low / medium / high)。当判断为 high 时,跳过第二段和第三段,直接输出"克制模式"暂停界面。

敏感场景包括但不限于:葬礼、病房、严重冲突、自杀相关、灾难现场。

## 1.5 功能模块详解

### 1.5.1 核心模块:三段式 Pipeline

**第一段 / 场景理解**

- 模型:待定（需要API接口时提醒我）
- 输入:base64 编码的暂停帧图片
- 输出 JSON 结构(详见 §9 Prompt 模板)

**第二段 / 广告决策**

- 模型:待定（需要API接口时提醒我）
- 输入:场景 JSON + 当前用户画像 + 当前用户状态 + 广告库 JSON
- 输出:品牌、文案、声线、契合度评分、决策理由

**第三段 / 原生广告生成**

- 模型:待定（需要API接口时提醒我）
- 两种生成模式:
    - **模式A(基础)**:文生图,生成一张与暂停帧色调统一的广告卡片,叠加在暂停帧角落
    - **模式B(进阶,做一个示例镜头)**:image edit + mask,在暂停帧的"空白表面"位置植入产品(VPP 虚拟产品植入)

### 1.5.2 对话式追问模块

- TTS 服务:ElevenLabs （需要API接口时提醒我）
- 声线系统:预设 4-6 个"情境声线",**严禁克隆任何剧中角色或真人声音**
    - 深夜电台型:低沉、缓慢、温暖(适合都市夜景、孤独场景)
    - 晨间咖啡馆型:明亮、轻盈、女声(适合甜宠、温馨场景)
    - 古风旁白型:苍劲、有韵律(适合古装、武侠场景)
    - 悬疑低语型:克制、紧绷、略带气声(适合悬疑、犯罪场景)
    - 冷峻陈述型:中性、几乎无情感色彩(适合科幻、商战场景)
- 声线选择由第二段 LLM 决定,不依赖用户主动选择

### 1.5.3 决策面板模块

右侧面板实时展示:

- VLM 输出的场景 JSON(折叠展示)
- 选中品牌 + 备选品牌(展示权衡过程)
- 决策评分:场景情绪契合度 / 品牌调性匹配度 / 用户当前心境干扰度 / 预测注意力捕获率
- 选中声线 + 选择理由

**重要**:所有评分由 LLM 在生成广告时同步输出,不需要真实算法支撑,但要展示得像一个严肃的决策系统。

### 1.5.4 用户画像切换器

- 预设 3-4 个画像,每个画像是一个 JSON 配置(年龄、性别、城市、消费偏好、禁忌品类)
- 切换画像后,第二段的 prompt 自动注入新画像
- UI 形态:顶部的下拉选择器或标签按钮

### 1.5.5 用户状态切换器

- 预设 2 个状态:情绪正常 / 情绪疲劳累积
- 切换状态后,第二段的 prompt 自动注入新状态上下文
- 状态B 时,LLM 应输出"不推广告,推荐切换内容"的决策

### 1.5.6 广告主入口(可选,时间充裕再做)

- 静态页面即可,证明产品的 SaaS 形态愿景
- 表单字段:品牌名、产品图、品牌调性描述、目标人群、目标场景关键词
- 提交后将该品牌追加到广告库 JSON 中(前端假数据持久化即可)

## 1.8 关键 Prompt 模板

### 1.8.1 第一段:场景理解 Prompt

```
你是一位专业的影视画面分析师。请分析下面这一帧视频画面,输出严格符合以下 JSON 结构的描述:

{
  "scene": "场景类型,如'都市夜晚居酒屋''古代集市''雨夜街头'",
  "characters": "画面中的人物状态,如'男性独自饮酒''两人争执'",
  "emotion": "场景的情绪基调,从['孤独','低落','紧张','甜蜜','悲伤','沉重','轻松','悬疑']中选1-2个",
  "color_tone": "色调描述,如'暖黄色低饱和''冷蓝色高对比'",
  "lighting": "灯光风格,如'昏暗温暖''强光高反差'",
  "objects": ["画面中的关键物体列表"],
  "empty_surfaces": ["画面中可用于产品植入的空白表面,如'吧台右侧空位''桌上空杯子'"],
  "sensitivity": "敏感度等级,从['low','medium','high']中选,葬礼/病房/严重冲突/自杀场景必须为high"
}

请只输出 JSON,不要任何额外说明。
```

### 1.8.2 第二段:广告决策 Prompt

```
你是腾讯视频的智能广告决策系统。基于以下信息做出决策:

【当前场景】
{stage1_output_json}

【用户画像】
{persona_json}

【用户当前状态】
{user_state}  // "normal" 或 "emotional_fatigue"

【广告库】
{ad_library_json}

请按以下规则决策:
1. 如果场景 sensitivity 为 "high",输出 {"decision": "no_ad", "reason": "..."}
2. 如果用户状态为 "emotional_fatigue",不出广告,改为推荐内容切换,输出 {"decision": "content_switch", "suggested_content": "...", "reason": "..."}
3. 否则,输出完整广告决策:

{
  "decision": "show_ad",
  "selected_brand": "选中品牌名",
  "alternative_brands": ["备选品牌1", "备选品牌2"],
  "ad_copy": "不超过20字、像电影旁白而非促销的广告文案",
  "voice_preset": "从['深夜电台型','晨间咖啡馆型','古风旁白型','悬疑低语型','冷峻陈述型']中选1个",
  "scores": {
    "scene_emotion_fit": 0-100,
    "brand_tone_match": 0-100,
    "user_disturbance": "low/medium/high",
    "predicted_attention_lift": "+XX% vs 传统弹窗"
  },
  "reasoning": "为什么选这个品牌、这个声线、这个评分的简短理由(中文,2-3句)"
}

只输出 JSON,不要任何额外说明。
```

### 1.8.3 第三段:图像生成 Prompt(模式A:原生广告卡片)

```
生成一张原生广告卡片,要求:
- 整体色调:{color_tone}
- 灯光风格:{lighting}
- 内容主体:{selected_brand} 的产品
- 广告文案:{ad_copy}(中文,必须像电影旁白,使用克制字体精准渲染)
- 风格:极简、克制、不要任何促销元素(不要"特价""限时"等字样)
- 尺寸:1024x768,横向构图,左侧留白用于叠加在视频画面上
```

### 1.8.4 第三段:图像生成 Prompt(模式B:场景内植入)

```
(使用 image edit + mask API)
在 mask 标记的位置({empty_surface}),自然地放置 {selected_brand} 的产品。
要求:
- 光照方向、强度与原画面完全一致
- 物体的阴影、反射符合场景物理逻辑
- 产品标签上的文字必须清晰可读
- 不要修改 mask 之外的任何像素
```

### 1.8.5 对话式追问 Prompt

```
你是 {voice_preset} 风格的广告 AI 助手。当前广告:{selected_brand},文案"{ad_copy}"。
用户问题:{user_question}

回答要求:
- 语气与{voice_preset}一致(如深夜电台型则低沉缓慢、克制温暖)
- 不超过30字
- 不使用淘宝直播间式的高亢语气
- 如果用户表达购买意向,回复"已为你加入购物车",不要追加促销话术
```
