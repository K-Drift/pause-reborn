# RAS — Restrained Advertising Score(克制广告分)

> 一个 0-100 的产品自有指标,用一个数字回答:**这次广告决策有多"克制"?**

---

## 为什么需要这个指标

传统广告系统的 KPI 是 CTR、CVR、ROAS——它们假设"用户被打断 = 广告生效"。但 Pause Reborn 的产品立场恰恰相反:**好的暂停广告应该让用户感觉不到被打扰,但仍然在长期记忆里留下品牌印象**。

这套立场需要一个新的度量单位。RAS 就是 Pause Reborn 自有的、用一个数字概括"克制度 + 契合度 + 长期价值"的综合分。

每次 AI 决策(无论是出广告、不出广告、还是切剧推荐)都会产出一个 RAS,作为这次决策的"自我评估"。

---

## 公式

```
RAS = scene_emotion_fit × 0.30
    + brand_tone_match  × 0.30
    + (100 - disturb)   × 0.25     // disturb: low=10 / medium=50 / high=90
    + liftScore         × 0.15     // parse "+208%" → min(208, 200)/2 = 100
```

| 权重 | 维度 | 含义 |
|------|------|------|
| 0.30 | `scene_emotion_fit` | 广告与场景情绪的契合度(LLM 直接控制) |
| 0.30 | `brand_tone_match` | 广告与品牌调性的匹配度(LLM 直接控制) |
| 0.25 | `100 - disturb` | 克制度——干扰度的反向值 |
| 0.15 | `liftScore` | 预测注意力提升(预测值,权重最低) |

### 权重选择依据

- **契合 + 调性 共占 60%**:这两项是 LLM 直接控制、SPEC 反复强调的"懂场景 + 懂品牌"的工程化体现,应作为主导。
- **克制度占 25%**:承载产品哲学的内核——"克制"不是附加项,而是核心维度。
- **注意力提升占 15%**:这是预测值,不是测量值,权重最低以避免对未验证的数字过度依赖。

## 三种决策分支的 RAS 行为

| 决策类型 | RAS | 标签 | 环色 | 解释 |
|----------|-----|------|------|------|
| `show_ad` | 0-100 | `克制典范` / `得体` / `可接受` / `需要克制` | 绿(success) | 按公式计算 |
| `no_ad` / `restraint` | **100** | `克制满分` | 橙(warning) | 拒绝出广告 = 克制广告的最高表达 |
| `content_switch` | **—** | `—` | 灰(neutral) | 陪伴模式不是克制事件,不计 RAS |

### 为什么 `no_ad` = 100 分

这不是数学技巧,而是产品哲学的工程化表达:

> **克制广告的最高表达是「在合适的时刻拒绝出广告」**——比如葬礼场景、病房场景、严重冲突场景。系统能识别并选择沉默,本身就是产品价值的最直接证明。

把 `no_ad` 的 RAS 设为 100 + 橙色环(区别于 `show_ad` 的绿色),既给"克制"一个符号化的胜利条件,又在视觉上避免把"高质量广告"和"零广告"混淆。

### 为什么 `content_switch` 不计 RAS

陪伴模式(用户情绪疲劳累积时改为推荐切剧)是关怀事件,不是克制事件。如果给它一个 RAS 分,会稀释指标的语义边界。所以 `content_switch` 显示 `—`,并附"陪伴模式不计 RAS"提示。

---

## 评级阈值

| RAS | 标签 |
|-----|------|
| ≥ 85 | 克制典范 |
| ≥ 70 | 得体 |
| ≥ 55 | 可接受 |
| < 55 | 需要克制 |

阈值按场景分布手工拍:大多数甜宠场景配饮品的契合度天然较高(85+),所以"克制典范"是真正契合的天花板;55-70 是"勉强能用,但已经露出广告感";<55 应该被产品端识别并替换。

## 在产品里的位置

- **C 端(主体验)**:每次暂停后,AI Decision Console 顶部展示 RAS 圆环 + 4 项 breakdown,作为本次决策的"自我评估"。
- **B 端(品牌主 Dashboard)**:展示该品牌投放历史的"平均 RAS"作为核心 KPI 之一,以及 30 天 RAS 趋势线作为投放质量监控。

## 局限与未来

RAS 当前是**前向预测分**——所有维度都是 LLM 在生成决策时一次性给出。未来如果产品上线,应该补一个**后验 RAS**:用户暂停后实际行为(停留时长、是否跳过、品牌主动回忆率)反推真实的 RAS,与前向预测分对照,作为模型迭代的反馈信号。

那时候 RAS 就从"自我评估"升级为"双向校准的产品指标"。
